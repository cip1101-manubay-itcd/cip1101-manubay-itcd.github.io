Now, let's try playing with physical and logical tags!

So, first of all, if I wanted to emphasize something, for example this block of text, then I would probably use the physical tag b or the logical tag strong. Both are great for making boldfaces, but strong is much more intuitive.

If I wanted to italicize something, I could just use the i tag or simply use the em tag. This is great to use whenever you just want something emphasized. You can also use the cite tag to reference outside sources, or dfn tag to define something. There's also the

address tag
for contact informations. However, I like intuitiveness rather than speed, so I would rather use the em tag.

To underline something, I can use the u tag or the insert tag. The purpose of this is to "insert" text into a document that was already written before. The ins tag is easier to remember, so I'll use that one instead.

Monospaced texts are shown in fixed-width. There are multiple ways to write monospaced texts. For example, you can use the code tag for writing computer codes, or the kbd tag for writing keyboard inputs. You can also try the samp tag to show sample outputs. All of these are logical tags, but I would probably use the code tag more.

Strikethroughs show a horizontal line through the text. There are two ways to write them: either you use the s physical tag or the del logical tag. Strikethroughs are basically text that you want deleted, but still visible on the browser. Honestly, there's no point in doing this unless you're updating an old blog, or using it for sarcasm. I like using the del tag.
